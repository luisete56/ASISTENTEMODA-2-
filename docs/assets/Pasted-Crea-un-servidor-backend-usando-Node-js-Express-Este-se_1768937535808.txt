Crea un servidor backend usando Node.js + Express.

Este servidor forma parte de un Trabajo Fin de Grado (TFG) sobre un asistente de moda compuesto por:
- Una app móvil
- Un servidor online central
- Un objeto físico inteligente situado en el armario

REGLAS OBLIGATORIAS DE ARQUITECTURA:
- El servidor es el único punto de comunicación entre la app y el objeto físico.
- El servidor contiene toda la lógica, estado y decisiones.
- La app y el objeto físico NO contienen lógica de negocio.
- El servidor expone una API REST que devuelve únicamente JSON.
- No incluir frontend complejo ni vistas dinámicas (solo API + un frontend informativo simple).

PARTES QUE DEBE TENER EL SERVIDOR:

1️⃣ Integración con ChatGPT (API + reglas) — MOCK
- Crear un módulo de integración con ChatGPT.
- De momento NO usar la API real.
- Implementar una función simulada que devuelva respuestas de ejemplo.
- La integración real con ChatGPT se hará más adelante usando variables de entorno.

Endpoint:
POST /chat

Respuesta simulada:
{
  "type": "text" | "image",
  "content": "string"
}

---

2️⃣ Base de datos del inventario (mock/simple)
- Almacenar prendas.
- Guardar:
  - nombre
  - imagen de referencia
  - embeddings (placeholder)
  - estado: dentro / fuera
  - imagen del cuerpo del usuario
- Puede usarse un archivo JSON o memoria en servidor.

Endpoints:
GET  /wardrobe
POST /wardrobe

---

3️⃣ Lógica de visión artificial del armario — MOCK
- Endpoint para recibir eventos del objeto físico.
- Recibe imágenes simuladas.
- Simula la comparación de prendas.
- Actualiza el estado del inventario.

Endpoint:
POST /wardrobe/event

Respuesta:
{
  "action": "none"
}

---

4️⃣ Lógica de generación de imágenes — MOCK
- Simula la generación de looks.
- Devuelve una imagen de ejemplo (URL o placeholder).
- Esta imagen se envía a la app como respuesta del chat.

---

5️⃣ Frontend informativo del servicio
- Página web muy simple (HTML/CSS).
- Explica qué es el asistente de moda.
- Muestra:
  - el objeto físico
  - la app
  - el sistema como conjunto
- No tiene lógica ni acceso a datos.
- Solo sirve para presentar el proyecto de forma clara y visual.

---

REQUISITOS TÉCNICOS:
- Node.js + Express
- Estructura de carpetas clara:
  - routes
  - services
  - data
- Uso de JSON para todas las respuestas
- Código claro y comentado
- Preparado para sustituir los mocks por IA real más adelante

OBJETIVO:
Construir el servidor central de un sistema distribuido de asistente de moda, diseñado para integrar app móvil, objeto físico e inteligencia artificial sin fricciones.
